Grundstruktur

public class Main {
public static void main(String[] args) {
 System.out.println("Hello, World!");
 }
 }

Die Grundstruktur eines Java Projektes kann man grundsätzlich in 3 Teile gliedern.
Um den Code leichter verständlich zu machen, brechen wir den Code auf die Einzelteile herunter.

public class Main {...}

Diese Zeile besagt im Grunde das eine neue Klasse definiert wird. Eine Klasse ist im Grunde genommen eine Funktion, nur das man einer Klasse keine Parameter geben kann. Auch kann man in einer Klasse Daten von dem Rest des Codes abkapseln (siehe Kapitel Klassen).
Das KeyWord "public" wird dazu verwendet, dass man auf diese Klasse außerhalb der Datei, in der die Klasse definiert wurde, zugreifen kann. Auch kann man das Keyword "private" verwenden um die Funktion außerhalb der Datei als nicht zugreifbar zu markieren.
Also ein leeres Gerüst würde so aussehen:

public class Name {...}

Um nun zum Zweiten Part des Hello World Programmes zu kommen, müssen wir uns den Funktionen in Java widmen.

public static void main(String[] args) {...}

Hier wird wieder wie in der Klasse entweder das Keyword "public" oder "private" verwendet. Jedoch werden hier Zwei weitere neue Keywords hinzugefügt.
Zum einen gibt es das Keyword "static", welches für einen Zugriff auf die Funktion ermöglicht, ohne sich mit umständlichen Strukturen herumschlagen zu müssen.
Auch wird ein Datentyp in der Funktion angegeben. Hier wird nichts neues verwendet, was wir nicht schon aus der Programmiersprache C können. Das Keyword "void" steht einfach nur dafür, das nichts zurückgegeben wird.
Um nun ein "Hello World" in Java ausgeben zu können, brauchen wir bestimmte Parameter in der Main Funktion. Diese Parameter bestehen aus einem String-Array (siehe Kapitel Arrays) und einem Platzhalter für Argumente (siehe Kapitel Funktionen). Diese werden gebraucht um einen String, eine Aneinanderreihung von Zeichen, ausgeben zu können.

Eine Funktion besteht in Java aus der Öffentlichkeit der Funktion (public oder private), ob die Funktion static ist oder nicht, einem Datentypen, einem Namen und Parametern.

public static Datentyp Name(Parameter) {...}

Um nun einen Text ausgeben zu können, muss man einen bestimmten Befehl ausführen:

System.out.println("Hello, World!");

Um nun zu verstehen wie dieser Seltsame und lange Befehl zustande kommt, muss man wissen wie man in Java Funktionen aufruft.
In Java ruft man Funktion auf indem man zuerst den Namen der Library vor gibt. In diesem Fall währe das "System".
Danach muss man die Klasse angeben, in der die Funktion "println" liegt angeben. Hier währe das "out".
Nun muss man nur noch die Funktion angeben. In diesem Fall währe das "println", welches Kurz für "print line" steht. Bestimmte IDE's haben auch Shortcuts dafür. IntelliJ (die IDE von Jetbrains für Java) verwendet den Shortcut "sout".
