Verzweigungen

Verzweigungen erlauben es uns, unseren Code nur unter bestimmten Umständen auszuführen. Verzweigungen gibt es in den folgenden Ausführungen:

Einfach Verzweigung

Eine Einfach Verzweigung besteht aus einer einzigen if-Anweisung. Und ist wie folgt aufgebaut:

if (Bedingung){
...
}

Die Bedingung wird auf ihren Wahrheitswert geprüft (true/false) und besteht aus mind. 2 Werten und 1 logischem Operator (==, !=, <, >, <=, >=, siehe Kapitel 2).

Ist die Bedingung wahr, wird der sog. then-Zweig ausgeführt.

Zweifach Verzweigung

Die Zweifach Verzweigung ähnelt der Einfachen, jedoch ist diese durch eine else-Anweisung erweitert:

if (Bedingung){
...
}
else{
...
}

Ist die Bedingung wahr wird wie gewohnt der then-Zweig ausgeführt, sollte nun die Bedingung aber falsch sein, so wird der else-Zweig ausgeführt.

Mehrfach Verzweigung - If

Ein If-Else Gerüst lässt sich nun weiter in ein If-Else If-Else Gerüst erweitern, aus diese Weise lassen beliebig viele Bedingungen prüfen. Die minimalistische Schreibweise wäre:

if (Bedingung 1){
...
}
else if (Bedingung 2){
...
}
else{
...
}

Hierbei ist zu beachten, dass die Bedingungen von oben nach unten geprüft werden, nur einer der Zweige kann ausgeführt werden. Sobald eine Bedingung wahr ist, werden die weiteren Bedingungen nicht mehr überprüft.

Mehrfach Verzweigung - Switch

Die Switch-Verzweigung kann, im Gegensatz zur If-Verzweigung, nur die Übereinstimmung numerischer Werte überprüfen. D. h. die Logischen Operatoren kommen hier nicht zum Einsatz. Die Switch-Verzweigung wird zudem von vielen Menschen lesbarer empfunden als ein If-Else If-Else Gerüst:

switch(Variable) {
case Zahl 1:
...
break;
case Zahl 2:
...
break;
default:
...
}

Zu beachten sind sie break-Anweisungen in den case-Zweigen, diese sorgen dafür, dass nach der Ausführung des case-Zweigs kein weiterer Zweig ausgeführt wird. Die break-Anweisung ist im default-Zweig nicht notwendig.

Die case-Anweisungen sind Vergleichbar zu den if und if else-Anweisungenen, während die default-Anweisung mit der else-Anweisung übereinstimmt.

Schleifen

Mittels Schleifen lässt sich ein Code-Block wiederholt ausführen, jede der 3 Schleifen verhält sich etwas unterschiedlich:

while-Schleife

Die while-Schleife erlaubt es uns, unseren Code so lange wiederholt auszuführen, solange die angegebene Bedigung erfüllt (true) ist. Die Bedingung wird vor der Ausführung des Code-Blocks überprüft. Aufgrund dieser Eigenschaft kann der Code 0 bis unendlich oft ausgeführt werden

while (Bedingung) {
...
}

do while-Schleife

Obwohl die Funktionalität dieser Schleife der while-Schleife ähnelt, ist der Aufbau einer do while-Schleife deutlich anders, wie sich folglich erkennen lässt. Die Bedingung wird erst nach der Ausführung des Code-Blocks überprüft. Eine do while-Schleife wird immer 1 bis unendlich-mal ausgeführt.

do {
...
} while (Bedingung);

for-Schleife

Der Aufbau einer for-Schleife unterscheidet sich deutlich vom Aufbau der anderen beiden Schleifen. for-Schleifen werden bei Zählabläufen verwendet oder wenn die Anzahl der Wiederholungen im Vorhinein bekannt ist:

for (Initialisierung; Bedingung; Aktualisierung) {
...
}

Bei der Initialisierung wird die sog. Laufvariable auf einen Startwert gesetzt, wie bei den vorherigen beiden Schleifen muss erneut die Bedingung erfüllt sein. Bei der Aktualisierung wird der Wert der Laufvariable geändert. Somit kann eine for-Schleife, genau so wie eine while-Schleife, 0-unendlich oft ausgeführt werden.

Info: Nur die Bedingung wird benötigt Initialisierung und Aktualisierung sind optional (ist aber nicht empfehlenswert).

Sonstiges

Im Bezug auf Schleifen gibt es ein weiteres relevantes Schlüsselwort:

  • break:
    Führt zum sofortigen Beenden einer Schleife.


Übungen

Füllen Sie die Lücken aus, um ein ausführbares Programm zu erzeugen:

include <stdio.h>

int main () {
int i = 0;
{
printf("Hello World\n");
i = i + 1
} while (i < 10)
}
