Der wesentliche Unterschied zwischen Unterprogrammen und Funktion ist, dass Funktionen einen Wert zurückschicken können und Unterprogramme nicht. Beide werden verwendet, wenn dieselbe Anweisung mehrmals benötigt wird, um die Lesbarkeit zu erhalten. Von weder einer Funktion noch einem Unterprogramm lässt sich auf Variablen zugreifen, die in der Main-Funktion definiert wurden, dafür sind Parameter oder sog. Globale Variablen notwendig (wobei Letzteres nicht empfohlen ist).

Es ist empfehlenswert komplexere Algorithmen bzw. Abläufe in einzelne Funktionen/Unterprogramme zu gliedern.

Unterprogramme

Ein Unterprogramm schickt keine Werte zurück, sondern führt nur eine Reihe von Anweisungen aus. Einem Unterprogramm kann man sog. Parameter angeben. Diese sind Werte, die die Funktion evtl. für ihre Verarbeitung benötigen könnte.

void Name(Datentyp Name, ...){
...
}

Zu beachten ist das void-Schlüsselwort, es gibt an das es sich hierbei um ein Unterprogramm handelt (Obwohl unter speziellen Umständen void auch ein 'Datentyp' ist, dies ist aber nicht relevant). Obwohl Unterprogramme keinen Wert zurückschicken, kann das return-Schlüsselwort genutzt werden, um das Unterprogramm frühzeitig zu beenden, dies erfolgt auf diese Weise:

void Unterprogrammname(Datentyp Parametername, ...){
...
return;
}

Funktionen

Im Gegensatz zu Unterprogrammen können Funktionen auch einen Wert zurückschicken. Auch Funktionen können Parameter haben.

Datentyp Funktionsname(Datentyp Parametername, ...){
...
return ...;
}
